volatile是jvm提供的轻量级的同步机制，具有三大特性：保证可见性、不保证原子性、禁止指令重排。

在了解三大特性之前，我们需要了解一下java的内存模型–JMM (Java Memory Model)，它是一种抽象的概念，并不真实存在。它描述的是一组规则或规范，定义的是程序里的各个变量的访问方式。

JMM中关于同步的三条规定：

  • 线程解锁前，必须把共享变量的值刷新回主内存
  • 线程加锁前，必须读取主内存的最新值到自己的工作内存
  • 加锁和解锁用的是同一把锁

图解：
图1
主物理内存中存在一个Student对象，它有一个属性age，初始值为15。存在t1和t2线程想要修改主内存中的age的值，但并无法直接修改主内存中的变量值，需要先将变量拷贝回自己的工作内存，先在自己的工作内存中进行操作。

图2
此时线程t1将自己工作内存中的变量值修改为12，接着就会将主内存中的值修改为12，同时此操作会被线程t2知晓，这也就是可见性的体现。

由于JMM操作的是实体是线程，而每个线程创建时JVM会为其创建一个工作内存（也就是栈空间）。工作内存是每个线程的私有数据区域，JMM规定所有变量都是储存在主内存中，即主内存是共享内存区域，所有线程都可以访问。

需要注意的是：线程对变量的操作都必须在自己的工作内存中进行，首先将变量从主内存拷贝到自己的工作内存，然后对变量进行操作，操作完成后再写回主内存。

因此，不同线程无法访问对方的工作内存，线程间的传值必须通过主内存来完成。

禁止指令重排的知识小结：  
1、volatile实现禁止指令重排优化，从而避免多线程环境下程序出现乱序执行现象；  
2、**内存屏障**(Memory Barrier)，又可以称为内存栅栏，它是一个CPU指令，起到保证特定操作的执行顺序和保证某些变量的内存可见性的作用；  
3、由于编译器和处理器都可以执行指令重排的优化指令，若在指令间加一条内存屏障，则可以高斯编译器和处理器，不论什么指令都不能和这条内存屏障指令进行重排序操作，也就是说可以通过内存屏障禁止在其前后的指令进行重排序；  
4、写内存屏障（Store Memory Barrier）：处理器将存储缓存值写回主存（阻塞方式）  
   读内存屏障（Load Memory Barrier）：处理器，处理失效队列（阻塞方式）  
5、内存屏障种类  
[图3](https://ae01.alicdn.com/kf/Hd5390602779c42869426934073c5a77dD.png)    
6、volatile读前插读屏障，写后加写屏障，避免CPU重排导致的问题，实现多线程之间数据的可见性  
7、对于处理器来说，内存屏障会导致cpu缓存的刷新，刷新时，会遵循缓存一致性协议:  
- 解锁时(lock)，jvm会强制刷新cpu缓存，导致当前线程更改，对其他线程可见;  
- 标记volatile的字段，在写操作时，会强制刷新cpu缓存，标记volatile的字段，每次读取都是直接读内存;  
- 编译器在final写操作后，会插入内存屏障，来禁止重排序，保证可见性。
